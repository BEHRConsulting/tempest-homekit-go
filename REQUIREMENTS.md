## Important Implementation Notes

### HomeKit Sensor Compliance
‚ö†Ô∏è **Critical**: Due to HomeKit's limited native sensor types, the **Pressure** and **UV Index** sensors use the standard HomeKit **Light Sensor** service for compliance. These sensors will appear in the Home app as "Light Sensor" with "lux" units, but actually display atmospheric pressure (mb) and UV index values. Users should ignore the "lux" unit label for these sensors as this is a HomeKit platform limitation.

### WeatherFlow API Integration

#### API Endpoints
- ‚úÖ **Stations**: `GET https://swd.weatherflow.com/swd/rest/stations?token={token}`
- ‚úÖ **Observations**: `GET https://swd.weatherflow.com/swd/rest/observations/station/{station_id}?token={token}`
- ‚úÖ **NEW: Custom Station URLs**: Support for any weather endpoint via `--station-url` flag
- ‚úÖ **NEW: Generated Weather API**: Built-in `/api/generate-weather` endpoint with Tempest API compatibility

#### Flexible Data Sources (New Architecture)
- ‚úÖ **Unified Processing**: Single data pipeline handles all weather sources
- ‚úÖ **API Compatibility**: Custom endpoints must return Tempest API-compatible JSON format
- ‚úÖ **Generated Weather Mode**: Built-in weather simulation with realistic patterns
- ‚úÖ **Backwards Compatibility**: `--use-generated-weather` flag still supported

#### UDP Stream (Offline Mode) - NEW in v1.5.0
- ‚úÖ **Local Network Monitoring**: Listen for UDP broadcasts from Tempest hub on port 50222
- ‚úÖ **Offline Operation**: Monitor weather during internet outages without API access
- ‚úÖ **Real-time Updates**: Process observation messages broadcast every 60 seconds
- ‚úÖ **No API Token Required**: Complete local operation without WeatherFlow cloud services
- ‚úÖ **Message Types Supported**:
  - `obs_st`: Tempest device observations (18 fields: timestamp, wind, pressure, temp, humidity, lux, UV, rain, lightning, battery)
  - `obs_air`: AIR device observations (8 fields)
  - `obs_sky`: SKY device observations (14 fields)
  - `rapid_wind`: High-frequency wind updates
  - `device_status`: Battery, RSSI, sensor status
  - `hub_status`: Firmware, uptime, reset flags
- ‚úÖ **Configuration Flags**:
  - `--udp-stream`: Enable UDP broadcast listener
  - `--no-internet`: Disable all internet access (requires `--udp-stream`)
- ‚úÖ **Network Requirements**: Same LAN subnet, UDP port 50222 accessible
- ‚úÖ **Circular Buffer**: 1000 observation history with thread-safe access
- ‚úÖ **Web Dashboard Integration**: UDP status display with packet count, station IP, serial number
- ‚úÖ **Use Case**: Internet outage resilience - monitor weather when internet connectivity is unavailable

### HomeKit Implementation

#### Accessory Types
- ‚úÖ **Bridge**: `accessory.NewBridge` for device management
- ‚úÖ **Temperature Sensor**: `accessory.New` with `service.NewTemperatureSensor()` for air temperature
- ‚úÖ **Humidity Sensor**: `accessory.New` with `service.NewHumiditySensor()` for relative humidity
- ‚úÖ **Light Sensor**: `accessory.New` with `service.NewLightSensor()` for ambient light
- ‚úÖ **UV Index Sensor**: `accessory.New` with `service.NewLightSensor()` (custom range 0-15)
- ‚úÖ **Pressure Sensor**: `accessory.New` with `service.NewLightSensor()` (custom range 700-1200mb)

#### Standard Service Architecture
- ‚úÖ **Standard HomeKit Services**: Uses only built-in HomeKit service types for maximum compatibility
- ‚úÖ **Temperature Sensor**: Standard `service.NewTemperatureSensor()` for air temperature
- ‚úÖ **Humidity Sensor**: Standard `service.NewHumiditySensor()` for relative humidity  
- ‚úÖ **Light Sensor**: Standard `service.NewLightSensor()` for ambient light, UV index, and pressure
- ‚úÖ **Modern hap Library**: Uses `github.com/brutella/hap` v0.0.32 with context-based server lifecycle
- ‚úÖ **Configurable Sensors**: Sensor accessories created based on `--sensors` flag configuration

#### Service Characteristics (HomeKit Accessories Only)
- ‚úÖ **Air Temperature**: `CurrentTemperature` (float, Celsius) - standard HomeKit characteristic
- ‚úÖ **Relative Humidity**: `CurrentRelativeHumidity` (float, percentage) - standard HomeKit characteristic
- ‚úÖ **Ambient Light**: `CurrentAmbientLightLevel` (float, lux) - standard HomeKit light sensor characteristic
- ‚úÖ **UV Index**: `CurrentAmbientLightLevel` (float, UV index 0-15) - uses Light Sensor service for compliance
- ‚úÖ **Atmospheric Pressure**: `CurrentAmbientLightLevel` (float, mb 700-1200) - uses Light Sensor service for compliance

**Not Implemented in HomeKit** (Available in Web Dashboard Only):
- Wind Speed, Wind Gust, Wind Direction
- Rain Accumulation, Precipitation Type
- Lightning Count, Lightning Distance

**Note**: Custom characteristics exist in `custom_characteristics.go` but are not used in the current implementation. The application uses only standard HomeKit services for maximum compatibility.

#### Weather API Client Pattern
```go
func GetObservation(stationID int, token string) (*Observation, error) {
    url := fmt.Sprintf("https://swd.weatherflow.com/swd/rest/observations/station/%d?token=%s", stationID, token)
    resp, err := http.Get(url)
    if err != nil {
        return nil, err
    }
    defer resp.Body.Close()
    
    var response struct {
        Status struct {
            StatusCode int `json:"status_code"`
        } `json:"status"`
        Obs []Observation `json:"obs"`
    }
    
    if err := json.NewDecoder(resp.Body).Decode(&response); err != nil {
        return nil, err
    }
    
    if len(response.Obs) == 0 {
        return nil, fmt.Errorf("no observations available")
    }
    
    return &response.Obs[0], nil
}
```
